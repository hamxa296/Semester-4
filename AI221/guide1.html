Autonomous Rescue Robot

Reinforcement Learning Master Scenario & Interactive Quiz

Environment Description

The robot moves on a grid. Each state contains: Robot position \( (x,y) \), Battery level \( (b) \), Heat exposure level \( (h) \).

Actions: UP, DOWN, LEFT, RIGHT, SCAN, CHARGE, EXIT.

Stochasticity: Movement succeeds 80% of the time (20% slip). SCAN succeeds 90%. EXIT requires >20% battery.

PART A — Fundamental RL Understanding

Q1: Define the formal RL concepts for this scenario.+

State Space: The set of all possible situations. \( s=(x,y,b,h) \)

Action Space: \( A=\{\text{UP, DOWN, LEFT, RIGHT, SCAN, CHARGE, EXIT}\} \)

Reward Function: Numerical feedback per state \( R(s) \). e.g., Survivor +100, Crash -150.

Policy: Behavior strategy mapping states to actions \( \pi(s)=a \).

Return: Total discounted future reward \( G_t=R_{t+1}+\gamma R_{t+2}+\dots \).

Discount Factor (\( \gamma \)): Controls importance of future rewards (0 to 1).

Q-function \( Q(s,a) \): Expected return if starting in state \( s \), taking action \( a \), then acting optimally.

Environment: The world (walls, hazards) interacting with the robot.

Agent: The rescue robot.

Markov Decision Process: The full framework: agent observes state, chooses action, environment gives next state and reward.

Q2: Compute the return \( G_0 \) for rewards {-1, -1, -20, 100, 200} with \( \gamma = 0.9 \).+
\[ G_0 = R_1 + \gamma R_2 + \gamma^2R_3 + \gamma^3R_4 + \gamma^4R_5 \] \[ G_0 = -1 + 0.9(-1) + 0.9^2(-20) + 0.9^3(100) + 0.9^4(200) \] \[ G_0 = -1 - 0.9 - 16.2 + 72.9 + 131.22 = 186.02 \]

Physical Meaning: Even though the robot suffered penalties initially, the later large rewards dominate. The sequence was highly successful.

Q3: Explain exactly how behavior changes if \( \gamma = 0 \).+

If \( \gamma = 0 \), then \( G_t = R_{t+1} \). The robot ONLY cares about immediate reward and becomes completely short-sighted. It would avoid immediate penalties but ignore future consequences, failing at long-term rescue planning.

Q4: What is one major advantage and disadvantage of \( \gamma = 0.999 \)?+

Advantage: Strongly values long-term outcomes. Can sacrifice small short-term reward for huge future mission success.

Disadvantage: Learning becomes harder/slower because extremely distant rewards heavily influence decisions (credit assignment becomes difficult).

PART B — Policies and Q-functions

Q5 & Q6: Write the optimal policy action mathematically for State s, and explain what \( Q(s,\text{SCAN})=70 \) means.+

Q5: Choose action with highest Q-value. \( \pi(s) = \arg\max_a Q(s,a) = \text{SCAN} \).

Q6: It means: If the robot starts in state \( s \), performs SCAN once, then behaves optimally afterward, the expected discounted future return is 70. It does NOT mean immediate reward is 70.

Q7 & Q8: Does Q-function measure immediate reward only? Why might SCAN be negative despite a +20 immediate reward?+

Q7: FALSE. It measures immediate reward PLUS discounted future rewards.

Q8: If the immediate reward is +20, but the action traps the robot leading to a -150 penalty later, the future negative consequences outweigh the immediate gain. Total return evaluates complete future consequences.

Q9: Explain the difference between \( Q^\pi(s,a) \) and \( Q^*(s,a) \).+

\( Q^\pi(s,a) \): Value assuming the robot follows a PARTICULAR policy (e.g., cautious vs aggressive) afterward.

\( Q^*(s,a) \): Optimal possible value. Assumes robot behaves perfectly after taking action \( a \).

PART C — Bellman Equation

Q10, Q11, Q12: Compute \( Q(s,a) \) using Bellman. Interpret it. What does "learning" mean here?+

Q10 Compute: Given \( R(s)=-1 \), \( \gamma=0.9 \), and \( \max Q(s',a')=60 \).
\( Q(s,a) = R(s) + \gamma \max_{a'} Q(s',a') = -1 + 0.9(60) = 53 \).

Q11 Meaning: The action is good overall because despite a small immediate penalty, it leads to excellent future states.

Q12 Learning: If the old belief was 120 and the new target is 53, the robot realized "This action is not nearly as valuable as I thought." Learning is correcting value estimates using experience.

PART D — Stochastic Environments

Q13 & Q14: Compute Expectation and Stochastic Bellman Estimate.+

Q13 Expectation:
\( E[\max Q(s',a')] = 0.5(100) + 0.3(-50) + 0.2(20) = 50 - 15 + 4 = 39 \).

Q14 Bellman:
\( Q(s,a) = R(s) + \gamma E[\max Q(s',a')] = -1 + 0.9(39) = 34.1 \).

Q15: Why is expectation REQUIRED in stochastic environments?+

Because the same action can lead to multiple outcomes. Ignoring it would overestimate lucky outcomes and underestimate risky outcomes. Expectation averages all possible futures properly.

PART E — Continuous State Spaces

Q16 & Q17: Why is the state space continuous, and why are Q-tables impractical?+

Q16: Variables (position, velocity, angle) can take infinitely many values, not just discrete labels.

Q17: Continuous variables create infinitely many possible states. You cannot store infinite table entries.

Q18: Compare this robot problem to Lunar Lander.+
  • Continuous Variables: position, velocity, angle, angular velocity.
  • Discrete Actions: fire left/right thruster, main engine, do nothing.
  • Reward Shaping: Positive for landing/grounded legs; Negative for crashing/fuel usage.

PART F — Deep Conceptual Questions

Q19: Does policy cause Q-function, or does Q-function cause policy?+

Both are partially correct. Policy influences Q-values because future behavior determines future rewards. Q-values influence policy because policy often chooses the highest-Q action. They improve each other iteratively.

Q20 & Q21 & Q22: Battery delay, Supervised vs RL, and value of crashes.+

Q20 Battery: Immediate +10, later -200. Total future return becomes highly negative. RL learns long-term consequences dominate.

Q21 RL vs Supervised: Supervised is given correct answers directly. RL has no correct action labels; the agent must discover good behavior via rewards and trial/error.

Q22 Crashes: Crashes generate strong negative rewards. They teach the agent "Actions leading here are bad." Bad experiences are informative training data.
