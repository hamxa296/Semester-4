PROCESSOR ARCHITECTURE

  • Program Counter
  • Instruction Memory
  • Register File
  • ALU & Control Unit
  • Data Memory
  • Imm Generator

Data Path & Control Flow Visualization

EXECUTION PIPELINE (SINGLE CYCLE)

FETCH → DECODE → EXECUTE → MEM → WB

The clock period is determined by the longest path (typically the LW instruction).

EXAMPLE: THE LOAD (LW) INSTRUCTION

lw x1, 0(x0)
  • Fetch instruction from Address 0
  • Decode Opcode and read Base Register (x0)
  • ALU computes Address: x0 + Offset
  • Data Memory reads value from that Address
  • Write-Back: Value stored into x1

TEST PROGRAM (ASSEMBLY)

lw x1, 0(x0) // Load 10 lw x2, 4(x0) // Load 20 add x3, x1, x2 // x3 = 30 sub x4, x2, x1 // x4 = 10 sw x3, 8(x0) // Store 30 at addr 8 and x5, x3, x4 // x5 = 10 or x6, x3, x4 // x6 = 30 beq x4, x1, 8 // Jump if equal (Taken) sw x1, 12(x0) // [SKIPPED] sw x2, 16(x0) // Final store

EXECUTION TRACE: INITIAL STATE

Starting with all Registers at 0

Memory[0] = 10 Memory[4] = 20

EXECUTION TRACE: RESULTS

Register File Updates:

x1 = 10 x2 = 20 x3 = 30 (10 + 20) x4 = 10 (20 - 10) x5 = 10 (30 & 10) x6 = 30 (30 | 10)

Memory & Branch:

Mem[8] = 30 BEQ (10==10) -> TAKEN Jump to PC 36 Mem[16] = 20
