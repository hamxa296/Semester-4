1AI Agents & Foundations

An agent perceives its environment through sensors and acts upon it through actuators. A rational agent chooses actions that maximise its expected performance measure.

PEAS Framework
  • Performance — what counts as success?
  • Environment — what world does the agent inhabit?
  • Actuators — what actions can it take?
  • Sensors — what can it perceive?
  • Example (self-driving car): P=safety, E=roads, A=steering/brakes, S=cameras/LIDAR

Traditional AI vs Modern AI

Traditional / Symbolic AI
  • Rule-based: explicit if-then logic
  • Expert systems (MYCIN, DENDRAL)
  • Brittle — breaks outside programmed rules
  • Transparent and interpretable
Modern / Statistical AI
  • Data-driven: learns from examples
  • Machine learning, deep learning, LLMs
  • Flexible — generalises to unseen data
  • Often a "black box"

2State Space & Problem Formulation
Five components of a search problem
  • 1. Initial state — where do we start?
  • 2. Actions — what can the agent do?
  • 3. Transition model — what state does action A in state S produce?
  • 4. Goal test — is this a solution?
  • 5. Path cost — cumulative cost of a sequence of actions

State vs. Node

State
  • Physical configuration of the world
  • Just the facts right now
  • No search metadata
Node (data structure)
  • Stores: state, parent, action, cost, depth
  • Multiple nodes can map to same state
  • Carries the journey, not just destination

State Space Graph vs. Search Tree

State Space Graph
  • Static map — each state appears once
  • Built conceptually before search
Search Tree
  • Generated dynamically during search
  • Same state may appear multiple times
State space graph vs search tree

4Informed Search & A*
h(n)
estimated cost from n to goal
Must never overestimate
Admissibility
  • h(n) ≤ h*(n) for all n — never overestimates true cost
  • With admissible h(n), A* is guaranteed OPTIMAL
  • Consistent: h(n) ≤ cost(n→n') + h(n') — needed for graph search optimality

A* Algorithm

A* formula
f(n) = g(n) + h(n)
Past cost + estimated future
A* expansion — nodes coloured by f(n) value. A* expands in f-cost order; greedy expands only by h(n)
A* Pros
  • Optimal with admissible h
  • Complete in finite spaces
  • Dominates all blind algorithms
A* Cons
  • High memory O(b^d)
  • Heuristic design is non-trivial

8-Puzzle Heuristics

Misplaced Tiles
  • Count tiles not in goal position
  • Weaker — h=1 even if 5 moves away
  • Expands more nodes
Manhattan Distance ← prefer this
  • Sum of row+col displacements
  • Tighter — h=5 if 5 moves away
  • Expands fewer nodes, faster
5Optimization

Only care about the quality of the final state — not the path. Navigate by moving between neighbouring states guided by an objective function.

Key vocabulary
  • Global optimum — absolute best solution
  • Local optimum — better than all neighbours, not globally best
  • Plateau — all neighbours have equal value — algorithm stalls
Optimization landscape — hill climbing vs simulated annealing

Hill Climbing

Always move to the best neighbouring state. Stop when no neighbour improves.

Pros
  • Extremely fast, near-zero memory
  • Works on smooth landscapes
Cons
  • Trapped in local optima
  • Stuck on plateaus and ridges

Simulated Annealing

Accept prob
P = e^(ΔE / T)
ΔE=change, T=temperature

High T → exploration (bad moves allowed). Low T → exploitation (only improvements). Cool gradually: T = T_max × α^t (α ≈ 0.99)

Pros
  • Escapes local optima
  • Used in VLSI, TSP, scheduling
Cons
  • Cooling schedule is an art
  • No finite-time guarantee
6Genetic Algorithm

Maintain a population of solutions and evolve them over generations.

GA cycle
  • 1. Initialise — N random solutions
  • 2. Evaluate — compute fitness f(s) for each
  • 3. Select — parents (higher fitness = higher chance)
  • 4. Crossover — combine parents → offspring
  • 5. Mutate — random gene changes
  • 6. Replace — offspring replace weakest
  • 7. Repeat until convergence
GA convergence — best and average fitness over generations

Selection Strategies

Roulette
P(i) = f(i) / Σ f(j)
Proportional to fitness
TypeHow it worksWhen to use
Single-point crossoverSplit at one point; swap tailsSimple problems
Two-point crossoverTwo cuts; swap middleMore mixing needed
Uniform crossoverEach gene from either parent 50/50Maximum mixing
Pros
  • Parallelisable population
  • Escapes local optima
  • Works on combinatorial problems
Cons
  • No global optimum guarantee
  • Many hyperparameters
  • Slow on smooth problems
7Adversarial Search

Zero-sum games: MAX maximises utility, MIN minimises it. Both play optimally.

Minimax tree with alpha-beta pruning — grey nodes are pruned (never evaluated)

Minimax Formulas

Terminal
MINIMAX(s) = UTILITY(s)
MAX node
max_a MINIMAX(RESULT(s,a))
MIN node
min_a MINIMAX(RESULT(s,a))

Alpha-Beta Pruning

Alpha (α)
best MAX can guarantee so far
MAX won't go below α
Beta (β)
best MIN can guarantee so far
MIN won't go above β
Alpha-Beta complexity
  • Best case (perfect ordering): O(b^(m/2)) — doubles searchable depth
  • Worst case (bad ordering): O(b^m) — same as plain minimax
  • Same result as minimax, always

Gradients shrink exponentially through many sigmoid/tanh layers. Fixed by: ReLU activations, Batch Normalisation, Residual connections (ResNet), Xavier/He initialisation.

12Case Studies

Read each scenario, pick your approach, then reveal the answer.

Case Study 1 — Warehouse Robot Navigation
Scenario

Fully-known warehouse grid, all moves cost 1, known start and destination, limited memory.

Which algorithm gives the optimal path with least wasted computation?
  • A) DFS — memory efficient
  • B) BFS — shortest path
  • C) A* with Manhattan distance
  • D) Genetic Algorithm
C) A* with Manhattan Distance
Fully observable, deterministic, known goal → perfect for A*. Manhattan distance is admissible on a grid. A* finds optimal path while expanding far fewer nodes than BFS. BFS works but is wasteful. DFS not optimal. GA is overkill.
Case Study 2 — Medical X-Ray Classification
Scenario

50,000 labelled chest X-ray images. Classify as healthy/pneumonia/COVID-19. Radiologists need to understand WHY decisions are made.

Which approach handles scale and meets interpretability requirements?
  • A) k-Nearest Neighbours
  • B) Single decision tree
  • C) CNN with Grad-CAM
  • D) SVM with RBF kernel
C) CNN with Grad-CAM visualisation
50,000 images → large-scale image task → CNNs are built for this. Grad-CAM highlights which image regions drove the prediction — satisfying radiologist interpretability. kNN and SVM cannot handle raw images at this scale. A single decision tree massively underfits pixel data.
Case Study 3 — Factory Job Scheduling
Scenario

Schedule 200 jobs across 50 machines to minimise completion time. NP-hard, billions of configurations, many local optima.

Which optimisation technique is best suited?
  • A) Hill Climbing
  • B) BFS
  • C) Genetic Algorithm
  • D) A* Search
C) Genetic Algorithm
Combinatorial NP-hard, exponential space, many local optima — the textbook GA use case. Each chromosome = job assignment permutation. Population diversity and crossover explore the space, escaping local optima that cripple hill climbing. BFS/A* are path-finding, not scheduling optimisers.
Case Study 4 — Chess-Like Game AI
Scenario

Strategy board game: branching factor 30, game length 40 moves, 10-second time limit per move.

How should the decision engine be designed?
  • A) Pure Minimax to depth 40
  • B) Alpha-Beta + iterative deepening + evaluation function
  • C) Genetic algorithm for move sequences
  • D) A* on the game tree
B) Alpha-Beta + iterative deepening + evaluation function
Pure minimax: 30^40 ≈ 10^59 nodes — infeasible. Alpha-Beta with good ordering reduces to ~30^20. Iterative deepening searches as deep as time allows and always has a move ready. Domain evaluation function replaces terminal utilities at the cutoff. GAs don't apply to sequential game decisions.
Case Study 5 — Customer Churn Prediction
Scenario

5,000 customers, 15 tabular features, binary label. Business team needs to understand WHICH factors drive churn to intervene.

Which model fits data size and interpretability requirements?
  • A) CNN
  • B) Decision Tree or Logistic Regression
  • C) kNN with k=1
  • D) Deep neural network (10 hidden layers)
B) Decision Tree (or Logistic Regression)
Small dataset, tabular features, critical interpretability → decision tree. Produces human-readable rules like "If usage < 20 min/day AND plan=basic → 85% churn". CNNs are for images. kNN k=1 massively overfits. Deep NN is overkill and uninterpretable at this scale.
Case Study 6 — Earthquake Rescue Drone
Scenario

Drone explores unknown building. Layout unknown in advance. Battery limits exploration to depth 15. Goal location unknown — no heuristic available.

Which search strategy suits these constraints?
  • A) A* search
  • B) Depth-Limited Search (limit=15)
  • C) BFS
  • D) Simulated Annealing
B) Depth-Limited Search (DLS)
Unknown goal → no heuristic → no A*. Battery = hard depth limit of 15 → DLS. BFS exhausts memory in a large unknown building. If depth bound uncertain, IDDFS preferred. Simulated Annealing is for optimisation, not constrained exploration.
Case Study 7 — Neural Network Hyperparameter Tuning
Scenario

Optimise learning rate, batch size, layers, dropout. Objective non-differentiable w.r.t. hyperparameters. Each evaluation = 2 hours of training.

Which optimisation approach is practical given evaluation cost?
  • A) Gradient descent on hyperparameters
  • B) Genetic Algorithm or Bayesian Optimisation
  • C) Hill climbing
  • D) Grid search over all combinations
B) Genetic Algorithm (or Bayesian Optimisation)
Non-differentiable → no gradient descent. Evaluations expensive (2h each) → minimise number of evaluations. GA searches intelligently via population crossover. Bayesian Optimisation is the modern favourite for expensive black-box functions. Grid search with 4+ continuous hyperparameters requires thousands of evaluations.
